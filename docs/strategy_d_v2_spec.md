## 1. 목표 (Goal)

한국 주식 시장에서 **일봉 기준 "쌍바닥 + 장악형 양봉" 패턴이 완성된 시점**에 저가주를 매수하여,
**1~3 거래일 내 +3~5% 목표가 도달 시 청산**하는 단기 스윙 자동 탐지 시스템을 구축한다.

### 정량적 목표

| 지표 | 목표 |
|------|------|
| 승률 (Win Rate) | ≥ 55% |
| 평균 보유일 | 1.5 ~ 2.0 거래일 |
| 평균 실현 수익 (건당) | +2.5% (거래비용 반영 후) |
| MDD (Maximum Drawdown) | ≤ 12% |
| 연환산 수익률 | ≥ 25% |
| 월간 시그널 수 | 10~30건 |

## 3. 전략 설계 (Approach)

### 3.1 진입 조건 (Entry Rules)

**평가 시점**: 매 거래일 **장 마감 10분 전 (15:20)** 당일 종가 근처에서 판단.

원본 로스 카메론 4단계 조건을 **일봉 기준**으로 충족해야 함:

| 단계 | 조건 | 변수 |
|------|------|------|
| 1 | 당일 일봉 RSI(14) ≤ 30 | `rsi_oversold` |
| 2 | 최근 N일 내 연속 음봉 ≥ 3일 **또는** 하락 장악형 음봉 | `sell_pressure` |
| 2+ | 최근 N일 내 종가가 볼린저 밴드(20, 2) 하단 터치/이탈 | `bb_lower_breach` |
| 3 | 직전 거래일 또는 당일에 **상승 장악형 양봉** 출현 | `bullish_engulfing` |
| 4 | **쌍바닥 패턴 형성** (1차 바닥 후 2차 바닥) | `double_bottom` |
| 4+ | **2차 바닥이 볼린저 밴드 내부**에서 형성 | `second_bottom_inside_bb` |
| 4++ | **2차 바닥에서 양봉 확인** (당일 일봉이 양봉) | `today_bullish` |

**중요**: 2차 바닥 확인 봉은 반드시 **당일(진입일)** 또는 **직전 1거래일** 이내에 형성되어야 한다. 오래된 패턴은 이미 반등이 진행되어 진입 우위가 없음.

#### 3.1.1 쌍바닥 감지 알고리즘 (구체 명세)

```python
def detect_double_bottom(
    df: pd.DataFrame,          # 최근 30거래일 일봉
    swing_window: int = 3,     # swing low 감지: 좌우 3거래일보다 낮은 지점
    min_gap_days: int = 5,     # 두 바닥 간 최소 거리 (거래일)
    max_gap_days: int = 20,    # 두 바닥 간 최대 거리 (거래일)
    price_tolerance: float = 0.03,  # 두 바닥 가격 차이 허용 범위 ±3%
) -> Optional[Tuple[int, int]]:
    """
    Returns:
        (first_bottom_idx, second_bottom_idx) 또는 None
    """
    # 1) swing low 감지: 좌우 swing_window일보다 모두 낮은 지점
    swing_lows = []
    for i in range(swing_window, len(df) - swing_window):
        window = df['low'].iloc[i - swing_window : i + swing_window + 1]
        if df['low'].iloc[i] == window.min():
            swing_lows.append(i)

    if len(swing_lows) < 2:
        return None

    # 2) 가장 최근 2개 swing low가 유효한 쌍바닥인지 확인
    idx_1 = swing_lows[-2]
    idx_2 = swing_lows[-1]

    gap = idx_2 - idx_1
    if not (min_gap_days <= gap <= max_gap_days):
        return None

    price_1 = df['low'].iloc[idx_1]
    price_2 = df['low'].iloc[idx_2]
    if abs(price_2 - price_1) / price_1 > price_tolerance:
        return None

    # 3) 2차 바닥이 최근 2거래일 이내에 형성되었는지 확인
    if len(df) - 1 - idx_2 > 1:
        return None  # 너무 오래된 패턴

    return (idx_1, idx_2)
```

**파라미터 근거**:
- `swing_window=3`: 좌우 3일(전후 일주일) 최저점이면 의미 있는 swing — 노이즈 필터
- `min_gap_days=5`: 5일 미만은 같은 바닥으로 간주, 쌍바닥 아님
- `max_gap_days=20`: 20일 초과는 패턴 무효, 이미 반등이 멀리 진행됨
- `price_tolerance=0.03`: 두 바닥이 ±3% 이내여야 "같은 지지선" 판정

### 3.2 보조 확인 (Confidence Booster)

필수는 아니지만 confidence score에 가중치 부여:

| 조건 | 점수 가산 |
|------|-----------|
| 1차 바닥 거래량 ≥ 20일 평균 × 2.0 (패닉 셀 흡수) | +0.10 |
| 2차 바닥 거래량 < 1차 바닥 × 0.5 (매도 압력 소진) | +0.10 |
| **전일** 외국인 또는 기관 순매수 (당일이 아닌 전일 기준) | +0.15 |
| MACD 히스토그램 음→양 전환 또는 감소 완화 | +0.10 |
| 시장 전체(KOSPI 지수) 당일 양봉 (역추세 진입 회피) | +0.05 |

Base confidence = 0.55, 최종 confidence = min(base + 가산, 1.0).

**중요**: 장 마감 10분 전(15:20) 실행 시점에는 **당일 투자자별 수급이 아직 집계되지 않음** (pykrx 기준 장 마감 후 1~2시간 지연). 따라서 "전일 수급"만 활용하여 타이밍 모순을 회피한다. 실시간 당일 수급이 필요하면 KIS API `FHKST01010900` (종목별 투자자 매매동향)을 별도 조회해야 하나, v2 범위에서는 제외.

### 3.3 청산 조건 (Exit Rules)

**진입가** = 당일 종가 (15:20 시점 추정가, 실제 체결은 동시호가 또는 당일 시장가)

| 트리거 | 조건 | 실행 |
|--------|------|------|
| **1차 목표** | 장중 주가 ≥ 진입가 × 1.03 (+3%) | 즉시 시장가 매도 |
| **2차 목표** | 1차 실패 시 다음날 장중 ≥ 진입가 × 1.05 (+5%) | 즉시 시장가 매도 |
| **고정 손절** | 장중 주가 ≤ 진입가 × 0.975 (-2.5%) | 즉시 시장가 매도 |
| **갭다운 손절** | 익일 시가 ≤ 진입가 × 0.97 (-3% 갭다운) | 시초가 손절 |
| **시간 손절** | 진입 후 3 거래일 장 마감 도달 | 당일 종가 강제 청산 |

### 3.4 종목 필터 (Universe Filter)

대상: KOSPI + KOSDAQ 전종목 중 아래 조건 전부 충족:

| 항목 | 기준 | 근거 |
|------|------|------|
| 시가총액 | 2,000억 ≤ × ≤ 30,000억 | 중소형주 변동성 최적 구간 |
| 20일 평균 거래대금 | ≥ 100억 | 유동성 확보, 슬리피지 최소화 |
| 최근 20일 일평균 거래량 | ≥ 100,000주 | 호가 공백 방지 |
| 상장 경과일 | ≥ 120일 | 신규 상장 초기 변동성 제외 |
| 관리종목/투자유의종목 | 제외 | pykrx로 조회 |
| 우선주 | 제외 | 유동성 부족 |
| 당일 상/하한가 | 제외 | 매도 체결 리스크 |
| 당일 시초가 갭 | ≤ ±5% | 이상 급등락 제외 |

### 3.5 포지션 관리 (Position Management)

| 항목 | 규칙 |
|------|------|
| 최대 동시 보유 종목 | **5개** |
| 종목당 포지션 크기 | 총 자본의 15~20% |
| 현금 보유 비율 | 최소 10% 유지 |
| 동일 섹터 중복 | 최대 2개까지 |
| confidence 우선순위 | score 높은 종목부터 선착순 진입 |

#### 3.5.1 동시 진입 선택 규칙 (Tie-Breaker)

하루에 시그널이 여러 개 발생했을 때 진입 순서 결정:

```
1차 기준: confidence score 내림차순
2차 기준 (동점 시): 시가총액 내림차순 (더 큰 종목 우선, 유동성 리스크↓)
3차 기준 (여전히 동점): 당일 거래대금 내림차순
4차 기준: ticker 오름차순 (결정론적 보장)
```

#### 3.5.2 자금 부족 처리

| 상황 | 처리 |
|------|------|
| 보유 슬롯 < 5개 + 충분한 현금 | 상위 confidence 순 진입 (슬롯 충족까지) |
| 보유 슬롯 < 5개 + 현금 부족 | 진입 가능한 만큼만 (다음 종목 skip) |
| 보유 슬롯 = 5개 | **기존 포지션 교체 금지** — 신규 시그널 전부 skip |
| 동일 종목 기존 보유 중 | 신규 진입 불가 (피라미딩 금지) |
| 동일 섹터 2개 보유 + 3번째 시그널 | skip하고 다음 다른 섹터 종목으로 |

**피라미딩 금지 사유**: 단기 스윙에서 평단 낮추기는 손실 확대로 이어짐. 손절 규칙이 단일 진입가 기준이어야 일관성 유지.

## 6. 리스크 및 대안 (Risks & Alternatives)

### 6.1 식별된 리스크

| 리스크 | 영향도 | 대응 |
|--------|--------|------|
| **오버나잇 갭다운** | 높음 | 갭다운 -3% 즉시 손절 규칙, 포지션 사이징 20% 이내 |
| **장 마감 직전 매수 체결 실패** | 중간 | 장 마감 동시호가 주문 또는 다음날 시가 보완 진입 |
| **상한가 도달 시 매도 불가** | 낮음 | 상한가는 익절에 유리, 다음날 청산 수용 |
| **pykrx 데이터 지연/오류** | 중간 | 2개 소스 교차검증 (pykrx + KIS API 종가) |
| **백테스트 과최적화** | 높음 | Out-of-sample 검증, 파라미터 민감도 분석 필수 |
| **시장 체제 변화 (예: 급락장)** | 높음 | 시장 필터 추가 (예: KOSPI 200일선 이하면 진입 중단) |
| **동시 다종목 진입으로 현금 부족** | 중간 | confidence 내림차순 우선 진입 + 최소 현금 10% 유지 |

### 6.2 검토된 대안

| 대안 | 채택 여부 | 사유 |
|------|-----------|------|
| 15분봉 정밀 진입 추가 | ❌ | 1-3일 보유와 스케일 불일치, 오버나잇 갭 리스크 |
| 60분봉 병행 확인 | ❌ | 복잡도 대비 가치 낮음, 일봉만으로 충분 |
| 52주 저점 필터 추가 | ❌ | 원본 전략 철학 유지, RSI 30↓ 조건에 이미 반영 |
| MACD 다이버전스 필수화 | ❌ | 시그널 빈도 과소, 보조 가산점으로 강등 |
| 시가총액 30,000억 초과 제외 | ✅ | 대형주 일일 변동 부족, 1-3일 +3% 도달 어려움 |

## 8. 테스트 전략 (Test Strategy)

### 8.1 단위 테스트

- **지표 계산**: pandas-ta와 값 비교, 0.01% 이내 일치 확인
- **캔들 패턴 감지**: 인위적으로 만든 OHLC 시퀀스에 대한 정답 검증
- **진입 조건 엔진**: 7가지 조건 개별 on/off 테스트
- **청산 조건**: 각 트리거별 단위 테스트 (익절/손절/시간손절/갭다운)

### 8.2 통합 테스트

- 특정 종목/날짜에 대한 end-to-end 실행
- Phase 1 → Phase 2 → 시그널 생성까지 데이터 흐름 검증
- SQLite 상태 일관성 검증

### 8.3 백테스트

- **기간**: 2022-01-01 ~ 2025-12-31 (4년, ~1000 거래일)
- **데이터**: pykrx 일봉 OHLCV + 투자자별 수급
- **종목 풀**: 매일 유니버스 필터 재적용
- **거래 비용**: 왕복 0.25% (증권거래세 0.18% + 슬리피지 0.07%)
- **초기 자본**: 1,000만원
- **포지션 사이즈**: 20%씩 최대 5종목

**Out-of-sample 검증**:
- In-sample: 2022-01 ~ 2024-12
- Out-of-sample: 2025-01 ~ 2025-12
- 두 구간의 성과 지표 편차가 크면 과최적화 경고

### 8.4 파라미터 민감도 분석

아래 파라미터 각각을 ±20% 변동시켜 성과 지표 안정성 확인:
- RSI 과매도 임계값 (30 → 25, 35)
- 볼린저 밴드 표준편차 (2.0 → 1.8, 2.2)
- 목표 수익률 (3%, 5% → ±1%p)
- 손절 비율 (-2.5% → -2%, -3%)

성과가 특정 파라미터 조합에만 몰리면 과최적화 신호 → 재설계.

### 8.5 Paper Trading

- 기간: 최소 2주 (10 거래일)
- 실제 주문 미체결, 가상 매매 기록
- 시그널 발생 타이밍 정확성, 실제 체결 가능성 검증
- KIS API 모의투자 계정 활용 권장

## 9. 구현 시 주의사항

### 9.1 한국 시장 특수성 반영

- **장 시간**: 09:00~15:30, 장 마감 동시호가 15:20~15:30
- **진입 시점**: 15:20의 예상종가로 판단, 실제 체결은 15:30 동시호가
- **매도 주문**: 시장가 주문보다 지정가(종가 근처) + 시장가 fallback 권장
- **세금**: 2026년 현재 거래세 0.18% (KOSPI 0.15% + 농특세 0.15%, KOSDAQ 0.18%)
- **양도소득세**: 대주주 기준 (종목당 10억 이상) 해당 시에만 적용
- **상하한가**: ±30%, 도달 시 체결 불가 가능성

### 9.2 pykrx 사용 시 주의

- `get_market_ohlcv_by_date`: 일봉 조회 (최근 데이터는 T+1 반영)
- `get_market_cap_by_ticker`: 시총 조회 (장 마감 후 반영)
- `get_market_trading_value_by_investor`: 투자자별 수급 (장 마감 후 1~2시간 지연)
- **장 마감 10분 전 실행 시**: 수급 데이터는 전일 기준 → 당일 수급은 다른 소스 필요 (KIS API 실시간)
